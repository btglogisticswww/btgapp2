# Техническое задание на разработку логистической CRM/ERP платформы для компании БТГ Логистикс

## Общее описание проекта
Требуется разработать комплексную веб-платформу для управления логистическими операциями компании БТГ Логистикс (https://btg-logistics.eu/). Платформа должна объединять функции CRM, ERP и TMS для эффективного управления заказами на перевозку грузов, взаимодействия с исполнителями, документооборота и финансового учета.

## Бизнес-модель
Компания БТГ Логистикс работает как экспедитор: менеджеры компании принимают заказы на перевозку груза от заказчиков, затем логисты подбирают подходящих исполнителей (перевозчиков). Основной доход компании формируется как разница между стоимостью услуги для заказчика и затратами на услуги исполнителей. Также предоставляются дополнительные услуги.

## Основные модули системы

### 1. Модуль управления пользователями и правами доступа
- Многоуровневая система ролей: администраторы, менеджеры продаж, логисты, финансисты и т.д.
- Настраиваемые права доступа к разделам системы
- Персональные кабинеты с настраиваемыми дашбордами для разных ролей
- Система аутентификации и безопасности

### 2. Модуль управления заказами
- Форма создания и редактирования заказа на перевозку с детальной информацией:
  - Данные заказчика (из базы или новые)
  - Тип перевозки (комплектная, сборная, экспресс и т.д.)
  - Параметры груза (вес, объем, особенности)
  - Маршрут (пункты отправления и назначения)
  - Сроки доставки
  - Дополнительные требования
- Статусная модель заказа с отображением текущего этапа
- Отслеживание истории изменений по заказу
- Система оповещений при смене статуса
- Привязка документов к заказу

### 3. Модуль работы с исполнителями (перевозчиками)
- База данных исполнителей с категоризацией по регионам и типам транспорта
- Система запроса ставок от исполнителей
- Сравнение предложений от разных исполнителей
- Формирование и отправка заданий исполнителям
- Отслеживание выполнения заданий
- Рейтинговая система исполнителей

### 4. Модуль TMS (Transport Management System)
- Планирование маршрутов перевозки
- Оптимизация логистических цепочек
- Мультимодальные перевозки (комбинирование разных видов транспорта)
- Визуализация маршрутов на картах
- Отслеживание грузов в реальном времени
- Расчет оптимальных маршрутов доставки

### 5. Модуль финансового учета
- Формирование счетов заказчикам
- Учет платежей от заказчиков
- Расчеты с исполнителями
- Финансовая аналитика и отчетность
- Расчет маржинальности по заказам
- Бюджетирование и контроль расходов

### 6. Модуль документооборота
- Система генерации договоров и актов
- Хранение и управление документами
- Электронный документооборот
- Интеграция с системами ЭДО
- Шаблоны документов для разных типов заказов

### 7. Модуль аналитики и отчетности
- Настраиваемые дашборды для разных ролей
- Отчеты по эффективности работы
- Анализ прибыльности по направлениям, типам перевозок, клиентам
- Прогнозирование спроса
- Экспорт отчетов в различные форматы

### 8. Модуль CRM
- Управление клиентской базой
- История взаимодействия с клиентами
- Воронка продаж
- Календарь и планирование задач
- Интеграция с электронной почтой
- Система напоминаний

### 9. Модуль HRM
- Учет сотрудников
- Расчет KPI и эффективности сотрудников
- Управление задачами персонала
- Расчет мотивационной части зарплаты (по выполненным заказам)

### 10. Модуль коммуникаций
- Внутренний чат и обмен сообщениями
- Система уведомлений и оповещений
- История коммуникаций по заказам
- Автоматические уведомления по смене статусов

## Технические требования
1. Архитектура: SPA (Single Page Application) или PWA (Progressive Web Application)
2. Backend: Node.js с Express.js или NestJS
3. Frontend: React.js или Vue.js с использованием современных UI-библиотек
4. База данных: MongoDB для гибкой структуры данных или PostgreSQL для сложных связей
5. API: RESTful API для взаимодействия между клиентом и сервером
6. Интеграции:
   - Картографические сервисы (Google Maps или Yandex Maps)
   - Системы трекинга грузов
   - API служб доставки
   - Платежные системы
   - Системы электронного документооборота
7. Масштабируемость: архитектура должна поддерживать увеличение нагрузки и добавление новых модулей
8. Безопасность: защита данных, шифрование, контроль доступа
9. Отказоустойчивость: система логирования, бэкапы, восстановление после сбоев
10. Адаптивный интерфейс для работы на различных устройствах

## Общие требования к дизайну
1. Соответствие корпоративному стилю с использованием следующей цветовой схемы:
   - Основной цвет: бордово-красный (#8B0000 или схожий оттенок)
   - Вторичный цвет: черно-серый (#333333 или схожий оттенок, не полностью черный)
   - Дополнительные цвета для фона и элементов интерфейса: оттенки серого (#F5F5F5, #E0E0E0, #CCCCCC)
2. Адаптивность для работы на устройствах с разным размером экрана
3. Минималистичный, современный дизайн с акцентом на функциональность
4. Интуитивно понятная навигация с учетом разных ролей пользователей
5. Эффективная визуализация большого объема данных
6. Удобные дашборды с персонализацией под конкретные роли
7. Конструкция в виде админ-панели с левым фиксированным навигационным меню (должен сворачиваться и разворачиваться), верхним фиксированным хедером для оперативных уведомлений и правой выдвижной коммуникационной панелью (тоже фиксированная, под хедером, должна разворачиваться и сворачиваться). Основной контент - расположен между навигационным меню и коммуникационной панелью, адаптивно сжимается и расширяется, при сворачивании меню и/или коммуникационной панели.
8. Можно взять за основу конструкцию, приложенную в файлах

## Дополнительные требования
1. Многоязычность (русский, английский, немецкий)
2. Темная и светлая темы интерфейса
3. Возможность кастомизации интерфейса под нужды конкретных пользователей
4. Система обучения и подсказок для новых пользователей
5. API для интеграции со сторонними системами
6. Мобильное приложение или адаптивная версия для полевых сотрудников
